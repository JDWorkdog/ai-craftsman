# PRD (Product Requirements Document)

Create comprehensive Product Requirements Documents optimized for developer comprehension.

## What It Does

This prompt generates professional PRDs with:
- Clear problem statements and goals
- Well-formed user stories
- Detailed functional and non-functional requirements
- Success metrics and timeline
- Scope boundaries (what's NOT included)

## When to Use

- Starting a new feature or product
- Documenting existing functionality
- Communicating requirements to engineering
- Getting stakeholder alignment on scope
- Creating a foundation for sprint planning

## Key Features

- Asks strategic clarifying questions first
- Uses rapid-response format (1A, 2B, 3C)
- Targets junior developer comprehension level
- Includes 12 comprehensive sections
- Identifies open questions and unknowns

## PRD Structure

The generated PRD includes:

1. **Introduction/Overview** - What and why
2. **Problem Statement** - The pain point being addressed
3. **Goals and Objectives** - Measurable targets
4. **User Stories** - "As a... I want... so that..."
5. **Functional Requirements** - What it must do
6. **Non-Functional Requirements** - Performance, security, etc.
7. **Non-Goals** - Explicit scope boundaries
8. **Design Considerations** - UI/UX requirements
9. **Technical Considerations** - Constraints and dependencies
10. **Success Metrics** - How to measure success
11. **Timeline and Milestones** - Key dates
12. **Open Questions** - Remaining unknowns
